Modules are one of the most important parts of the Gink programming
language.

There are two ways to compile a Gink program.

i. Using a file path:

This method is for people who want to build something quickly. It does
not require a reqter file in the directory.

If a reqter file is present, the file will not be treated as a single
file.

$ gink run <path_to_file>

Use the above command to run a single file.

ii. Using a project:

More often, you will need to run a full project rather than an
individual file. For that, Gink uses reqter files.

Package managers in many languages are frustrating, so I tried to
build an approach that takes the best ideas from different languages.

To create a project, use:

$ gink init <name>

or

$ gink init

If you use `gink init` without providing a name, you will be prompted
to enter one.

You can also create a reqter file manually, but this is strongly
discouraged. The parser is much stricter with reqter files than with
other files. Even a trailing space can trigger an error. Since the
file is meant to be generated automatically, the parser handles it
with strict validation.

Note that module names must be more than 0 characters and no longer
than 256 characters.

Module names also cannot be keywords. All keywords are listed in:

"Chapter: Appendix - The Keywords"

A module name must start with an alphabetic character. The remaining
characters must be alphanumeric or underscores.

Any other character will be rejected by both the reqter file and the
compiler.

Each directory is treated as a module. A directory will only be
included in a project if it contains a reqter file.

If a main module exists in the project, it is treated as an executable
project.

If no main module exists, it is treated as a library project.
